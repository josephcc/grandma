## 網站架構總覽

網站包含**繁體中文**與**英文**兩個語言版本，每個語言版本有以下頁面：

| 頁面     | 中文網址             | 英文網址              |
| -------- | -------------------- | --------------------- |
| 首頁     | /tw/                 | /en/                  |
| 創辦緣起 | /tw/story            | /en/story             |
| 創辦人   | /tw/founders         | /en/founders          |
| 最新消息 | /tw/news             | /en/news              |
| 愛心捐款 | /tw/donate           | /en/donate            |

所有可編輯的內容都放在 GitHub 上的 `content/` 資料夾中：

```
content/
├── site.json          ← 網站設定（導覽列、捐款資訊、頁尾）
├── tw/                ← 繁體中文內容
│   ├── index.md          首頁
│   ├── mission.md        基金會宗旨（顯示於首頁）
│   ├── story.md          創辦緣起
│   ├── founders.md       創辦人
│   ├── news.md           最新消息
│   └── donate.md         愛心捐款
└── en/                ← English content
    ├── index.md          Home
    ├── mission.md        Mission
    ├── story.md          Our Story
    ├── founders.md       Founders
    ├── news.md           Recent News
    └── donate.md         Donate
```

## 如何進入編輯頁面

1. 登入 GitHub (https://github.com)
2. 進入本專案的 GitHub 頁面 (https://github.com/josephcc/grandma)
3. 點選 `content` 資料夾
4. 選擇要編輯的語言資料夾（`tw/` 或 `en/`）
5. 點選要編輯的檔案（例如 `news.md`）
6. 點選檔案右上方的**鉛筆圖示**進入編輯模式

## 編輯內容文字

每個 `.md`（Markdown）檔案的結構如下：

```markdown
---
title: "頁面標題"
description: "頁面描述，用於搜尋引擎顯示。"
---

這裡是頁面的正文內容...
```

### 重要規則

- **`---` 區塊**（最上方兩條橫線之間的內容）是頁面設定，包含標題和描述。可以修改文字內容，但**不要刪除 `---` 符號**，也**不要改變格式**（例如不要刪掉引號）。
- **`---` 區塊下方**的文字是頁面正文，可以自由編輯。

### 基本 Markdown 語法

| 想要的效果     | 寫法                       |
| -------------- | -------------------------- |
| 大標題         | `# 標題文字`               |
| 中標題         | `## 標題文字`              |
| 小標題         | `### 標題文字`             |
| 粗體           | `**粗體文字**`             |
| 項目清單       | `- 項目一`                 |
| 連結           | `[顯示文字](網址)`         |
| 分隔線         | `---`（獨立一行）          |

## 各頁面說明與編輯方式

### 首頁（index.md）

首頁的版面由程式碼控制，首頁會自動從以下來源組合內容：
- `mission.md` → 顯示在「基金會宗旨」區塊
- `story.md` → 顯示「創辦緣起」摘要
- `site.json` → 標語、按鈕文字等

因此，**編輯 `mission.md` 和 `story.md` 的內容就會自動更新首頁**。`index.md` 本身通常不需要修改。

### 創辦緣起（story.md）

直接編輯 `---` 下方的正文即可。

### 創辦人（founders.md）

可編輯創辦人的文字介紹。但創辦人的**照片和姓名卡片**是由程式碼控制的，如需修改照片或卡片樣式，請聯繫技術維護者。

### 最新消息（news.md）

詳見下方「[新增最新消息](#新增最新消息)」段落。

### 愛心捐款（donate.md）

頁面上半部的文字說明可以直接在 `donate.md` 中編輯。下半部的**捐款帳戶資訊**（銀行帳號、信用卡連結等）是從 `site.json` 讀取的，修改方式見「[編輯網站設定 site.json](#編輯網站設定-sitejson)」。

### 基金會宗旨（mission.md）

詳見下方「[編輯宗旨項目與圖示](#編輯宗旨項目與圖示)」段落。

## 新增最新消息

最新消息頁面（`news.md`）的每則消息格式如下：

```markdown
## 2026-03-15｜新聞標題

新聞內文內容在這裡。可以寫多行。

---
```

### 新增一則消息的步驟

1. 打開 `content/tw/news.md`（中文）或 `content/en/news.md`（英文）
2. 在 `# 最新消息` 標題下方（現有消息的**上方**），加入新的消息區塊：

```markdown
## 2026-04-01｜新的消息標題

消息內文寫在這裡。

---
```

3. 注意事項：
   - `##` 後面要有一個空格
   - 日期格式為 `YYYY-MM-DD`（西元年-月-日）
   - 日期與標題之間用全形「｜」分隔
   - 每則消息之間用 `---` 分隔線隔開
   - **新消息放在最上方**，舊消息自然往下排

## 編輯宗旨項目與圖示

宗旨頁面（`mission.md`）中的項目使用 emoji 表情符號作為圖示，格式如下：

```markdown
- 🏥 強化重大疾病的診療資源與創新療法推廣
- 🔬 支援病人參與臨床研究與新興治療的機會
- 🧬 推動跨科別、跨領域的臨床與基礎醫學發展
- 💝 建立結合人文關懷與科學的整合醫療平台
```

### 修改方式

- **修改文字**：直接編輯 emoji 後面的文字即可
- **更換圖示**：將 emoji 替換為其他 emoji
- **新增項目**：在清單最後加一行 `- 🔹 新項目文字`
- **刪除項目**：直接刪除該行

### 可用的 emoji 圖示參考

| 圖示 | 含義     | 圖示 | 含義     |
| ---- | -------- | ---- | -------- |
| 🏥   | 醫院     | 💊   | 藥物     |
| 🔬   | 研究     | 🩺   | 診療     |
| 🧬   | 科學     | 🤝   | 合作     |
| 💝   | 關懷     | 🌱   | 成長     |

> 提示：emoji 可以從手機鍵盤複製貼上，或在 Mac 上按 `Control + Command + 空白鍵` 開啟 emoji 選單。

## 編輯網站設定 site.json

`content/site.json` 包含以下網站設定：

- 導覽列文字（首頁、創辦緣起、創辦人……）
- 首頁標語與按鈕文字
- 頁尾地址與版權聲明
- 捐款帳戶資訊（銀行帳號、信用卡連結、匯款資訊）
- 基金會 email

### 重要注意事項

修改 `site.json` 後，**需要技術維護者審核才能發布**。這是因為 JSON 格式如果寫錯（例如漏掉逗號或引號），會導致整個網站無法正常顯示。

### JSON 編輯基本規則

```json
{
  "欄位名稱": "欄位內容",
  "另一個欄位": "另一個內容"
}
```

- 所有文字必須用**英文雙引號** `"` 包住（不是中文引號「」）
- 每個欄位之間用**英文逗號** `,` 分隔
- **最後一個欄位後面不要加逗號**
- 不要刪除或修改欄位名稱（冒號左邊的部分），只修改欄位內容（冒號右邊的部分）

### 常見修改範例

**更新捐款連結：**
找到 `"url": "https://donate.newebpay.com/..."` 這行，把網址替換成新的即可。

**更新頁尾地址：**
找到 `"address": "..."` 這行，修改地址文字。

**更新 email：**
找到 `"email": "..."` 這行，修改 email 地址。

## 儲存與發布流程

### 修改中文或英文內容（content/tw/ 或 content/en/ 下的 .md 檔案）

1. 在 GitHub 上完成編輯後，點選頁面下方的 **「Commit changes...」** 按鈕
2. 在彈出的視窗中，可以簡單寫一下修改說明（例如「更新最新消息」），也可以不填直接送出
3. 選擇 **「Commit directly to the `main` branch」**，然後按 **「Commit changes」**
4. 儲存後，網站會**自動重新部署**，大約 **1～2 分鐘**後就會生效
5. 到 https://lhyymed.fund 重新整理頁面即可看到更新

> 中文和英文的 Markdown 內容檔案**不需要任何人審核**，儲存後直接生效。

### 修改網站設定（site.json）

1. 編輯完成後，點選 **「Commit changes...」**
2. 選擇 **「Create a new branch for this commit and start a pull request」**
3. 點選 **「Propose changes」** 建立 Pull Request
4. 等待技術維護者審核通過後，才會發布到正式網站

## 需要技術維護者協助的情況

以下修改**無法**透過編輯 Markdown 檔案完成，需要聯繫技術維護者：

| 需求                         | 原因                                       |
| ---------------------------- | ------------------------------------------ |
| 新增一個全新的頁面           | 需要建立新的頁面程式碼與導覽列設定         |
| 修改頁面版面配置或設計       | 版面由程式碼控制                           |
| 更換創辦人照片               | 圖片檔案需要透過技術流程上傳               |
| 更換網站主視覺背景圖         | 圖片檔案需要透過技術流程上傳               |
| 更換網站 Logo                | 圖片檔案需要透過技術流程上傳               |
| 修改網站顏色或字型           | 需要修改 CSS 樣式程式碼                    |
| 修改網域或 DNS 設定          | 需要在 Vercel 和域名管理介面操作           |
| 修改 site.json 格式錯誤      | JSON 格式問題需要技術者修復                |

## 常見問題

### Q: 我修改了檔案，但網站沒有更新？

- 請等待 1～2 分鐘，網站會自動部署。
- 嘗試**強制重新整理**網頁（Mac: `Command + Shift + R`；Windows: `Ctrl + Shift + R`）。
- 確認修改的是 `main` 分支（branch），而不是其他分支。

### Q: 我不小心改壞了檔案怎麼辦？

- 在 GitHub 上，進入該檔案的頁面，點選 **「History」** 查看歷史版本。
- 找到上一個正確的版本，點選該版本的 commit 標題進入。
- 可以看到之前的檔案內容，複製後重新編輯回正確的版本。
- 或者聯繫技術維護者協助還原。

### Q: 我可以同時修改中文和英文版本嗎？

可以。中文和英文是各自獨立的檔案，分別修改、分別儲存即可。建議兩邊的內容保持一致。

### Q: 新增消息時，需要同時更新中文和英文嗎？

建議同時更新，但不是強制要求。如果只更新其中一個語言版本，另一個語言版本只是不會顯示該則消息而已，不會出錯。

### Q: Markdown 檔案中的 `---` 是什麼意思？

`---` 在不同位置有不同含義：
- **檔案最開頭**的兩組 `---` 之間是**頁面設定**（title 和 description），請小心修改。
- **正文中**的 `---` 是**分隔線**，用來在視覺上區分不同段落。

### Q: 我能預覽修改後的效果嗎？

如果使用 Pull Request 流程（例如修改 `site.json` 時），Vercel 會自動產生一個預覽網址，可以在 Pull Request 頁面中看到連結。點選預覽連結即可查看修改效果。

對於直接 commit 到 main 的內容修改，則需等待 1～2 分鐘部署完成後，直接到 https://lhyymed.fund 查看。
